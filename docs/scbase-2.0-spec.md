# 超级细胞营地 2.0 改版 Spec

状态：v0.8，首页与游戏频道编辑网格改造完成，进入 AdSense 内容质量整改阶段
日期：2026-07-22
目标：把超级细胞营地从文章流型站点升级为超级细胞游戏中文资料与攻略中心，提升用户价值、搜索质量信号和 AdSense 审核通过率。

相关文档：[SCBase 运维手册](./scbase-operations.md)、[皇室战争卡牌与卡组系统 Spec](./scbase-clash-royale-cards-decks-spec.md)、[AdSense 内容质量审计](./adsense-content-audit-2026-07.md)

## 0. 已确认决策

- 站点中文名：统一使用“超级细胞营地”，`SCBase` 主要保留为域名和内部技术命名。
- 首页定位：超级细胞游戏中文玩家营地，第一阶段内容权重明显偏向皇室战争。
- URL 结构：已新增 `/clashroyale/`、`/brawlstars/`、`/clashofclans/`、`/moco/`、`/tools/` 顶级入口；旧文章 URL 暂不迁移。
- 构建产物：`public/` 已从 Git 跟踪中移除并加入 `.gitignore`；原先需要保留的 `ads.txt`、`about.webp`、`kl/` 工具资源已迁移到 `static/`。
- Phase 1/2 基础改造：信任页、首页、栏目页、文章页基础模板、工具页、结构化数据和新文章 archetype 已初步落地。
- 审核状态：AdSense 未通过，原因是复制内容页面展示广告和低价值内容；当前按审计结果收紧广告范围并处理过期薄内容。
- 视觉方向：首页和四个游戏频道已完成编辑网格改造；全站采用适中的中文字号和字重，正文优先保证阅读舒适度，标题通过层级、留白和主题色短线建立重点。
- 内容标准：之后凡是由 Codex 参与整理、创作或改写的超级细胞营地文章，默认同时按公众号阅读体验和 AdSense/搜索质量标准把控。

## 1. 背景

当前站点主要以 Hugo 文章发布为核心，内容覆盖皇室战争、荒野乱斗、部落冲突、Mo.Co 等 Supercell 游戏。现有结构偏博客流，首页和栏目页对内容价值的组织较弱，Google 可能更容易把它识别为低原创度、薄内容、重复归档页面较多的站点。

超级细胞营地 2.0 的目标不是单纯换 UI，而是重构内容表达方式：

- 从“发布文章”转向“沉淀可长期引用的攻略与资料”。
- 从“最新内容列表”转向“游戏入口 + 专题中心 + 工具/资料入口”。
- 从“搜索引擎抓到大量薄页面”转向“抓到少量但结构清晰的高价值页面”。

## 2. 产品定位

超级细胞营地是面向中文玩家的超级细胞游戏攻略、资料与工具站。

第一阶段以皇室战争为主站点核心，其他游戏保留入口但降低薄页面暴露：

- 皇室战争：主力内容，优先建设栏目、专题、工具和长期攻略。
- 荒野乱斗：保留新闻和攻略入口，后续补齐资料页。
- 部落冲突：保留栏目入口，聚合更新与活动。
- Mo.Co：保留观察和资讯入口。

## 3. 成功指标

短期目标：

- 站点首页能在 10 秒内让新用户理解“这里能帮我解决什么问题”。
- 皇室战争栏目页不再只是文章列表，而是可导航的攻略中心。
- 新文章具备统一的信息增量结构，避免纯资讯感。
- 补齐 AdSense 常见信任页：关于、联系方式、隐私政策、版权声明、免责声明。

中期目标：

- Google Search Console 中低质量/已抓取未编入页面比例下降。
- 核心页面获得更多自然搜索入口。
- 用户从首页进入文章或专题的路径更清晰。

## 4. 信息架构

### 4.1 顶部导航

建议导航：

- 首页
- 皇室战争
- 荒野乱斗
- 部落冲突
- Mo.Co
- 工具
- 关于

说明：

- 不建议把标签、归档、全部分类直接暴露为主要入口。
- “工具”可以先是占位栏目，后续接卡组推荐、活动日历、版本追踪等。

### 4.2 首页

首页定位为统一资讯入口，游戏百科、攻略和工具的详细功能继续放在各游戏二级页面。

当前结构：

1. 置顶内容
   - 左侧一篇主内容，右侧四篇次要内容。
   - 移动端主内容独占一行，四篇次要内容采用 2×2 宫格。
2. 热门内容
   - 横向轮播，支持分页、循环、键盘焦点暂停和减少动画设置。
3. 最新资讯
   - 桌面端 3 列共 12 篇，移动端为左图右标题和日期的列表。
4. 常用工具
   - 站内查询工具与可靠的外部功能入口。
5. 游戏频道
   - 皇室战争、荒野乱斗、部落冲突和 Mo.Co 二级入口。

首页编排使用文章 Front Matter：

```yaml
home_pinned: 1
# home_popular: 1
```

- `home_pinned` 使用 1-5 控制置顶顺序。
- `home_popular` 使用正整数控制热门顺序。
- 同一文章不应同时配置两个字段。
- 配置不足时模板使用未重复的最新文章补位；最新资讯排除置顶和热门后取 12 篇。

### 4.3 皇室战争栏目页

皇室战争栏目页是第一阶段最重要的 SEO 和用户入口，当前结构已经落地：

1. 栏目头部
   - 标题：皇室战争攻略中心
   - 描述：卡组推荐、版本更新、活动挑战、新卡解析与实用指南。

2. 快捷入口
   - 最新文章
   - 卡组榜单
   - 卡牌百科
   - 免费奖励

3. 当前重点
   - 左侧一篇主内容，右侧四篇附稿。
   - 主内容标题和日期覆盖于图片底部；附稿采用紧凑缩略图列表。
   - 移动端主内容全宽，附稿改为左图右文。

4. 长期指南
   - 如何组卡
   - 卡组原型介绍
   - 超级选卡攻略
   - 新手升级建议

5. 最新文章列表
   - 桌面端最多 9 篇，采用 3 列编辑网格。
   - 移动端采用左图右文单列，隐藏摘要以控制页面高度。

荒野乱斗、部落冲突和 Mo.Co 使用同一套共享频道模板与各自主题色；由于长期内容储备不足，当前不强行增加“精选长期攻略”模块。

### 4.4 文章页

文章页保留现有 Hugo Markdown 工作流，但增加标准化内容模块。

建议每篇文章包含：

- 标题
- 更新时间
- 摘要
- 适合人群
- 正文
- 关键结论
- 相关攻略
- FAQ
- 资料来源或说明

Front matter 可逐步扩展：

```yaml
title:
description:
date:
updated:
image:
game: clashroyale
content_type: deck-guide
difficulty: beginner | intermediate | advanced
evergreen: true
featured: false
home_pinned: 1 # 与 home_popular 二选一
faq:
  - question:
    answer:
related:
  - /posts/...
```

第一阶段不要求所有旧文章补齐；AdSense 审核期间也不做旧文章批量重写。新文章应默认按此结构写。

## 5. 内容类型

### 5.1 长期攻略

长期攻略是超级细胞营地 2.0 的核心质量资产。

示例：

- 皇室战争如何构建一套合理卡组
- 皇室战争卡组原型介绍
- 超级选卡挑战攻略
- 新手升级资源分配指南

要求：

- 能在 3-12 个月后仍然有价值。
- 标题避免纯新闻口吻。
- 提供清晰步骤、判断标准和常见误区。

### 5.2 卡组推荐

卡组推荐要从“列卡组”升级为“解释为什么可用”。

建议结构：

- 适合玩家
- 卡组代码或卡牌列表
- 核心打法
- 起手建议
- 对局思路
- 可替换卡牌
- 优缺点

### 5.3 版本更新

版本更新不只是翻译公告。

建议结构：

- 更新重点
- 对普通玩家的影响
- 强势/弱势变化
- 建议优先做什么
- 相关攻略入口

### 5.4 资讯短文

资讯可以保留，但需要降低站点主体占比。

处理策略：

- 短资讯可以归入周报或专题汇总。
- 避免大量 300-500 字的独立薄页面。
- 无长期价值的旧资讯可考虑 noindex。

## 6. SEO 与 AdSense 改进

### 6.1 必备信任页

已初步完成：

- 关于我们
- 联系方式
- 隐私政策
- 版权声明
- 免责声明

### 6.2 薄页面处理

重点检查：

- 空分类页
- 低内容标签页
- 分页页
- 重复归档页
- 未命名页面

后续处理方案：

- 对低价值标签页加 `noindex,follow`。
- 只保留有真实专题价值的标签入口。
- 分类页增加栏目介绍和精选内容。
- 分页页减少在首页和导航中的权重。

当前状态：

- taxonomy 和 term 页面已统一使用 `noindex,follow`，并从 sitemap 排除。
- 第一批 14 篇过期预告、猜测、促销和短快讯已设置 `noindex`，保留旧 URL。
- 后续按内容专题逐批处理，不按篇幅机械删除文章。

### 6.3 结构化数据

优先添加：

- WebSite
- Organization
- Article
- BreadcrumbList
- FAQPage

### 6.4 内部链接

每篇新文章至少包含：

- 1-3 个相关站内文章链接。
- 1 个栏目页或专题页链接。
- 必要时链接到工具页。

## 7. 视觉与交互方向

整体方向：

- 少营销感，多资料站/攻略站感。
- 首页和栏目页要更像“可探索的目录”，不是单纯博客。
- 卡片信息密度适中，避免只有大图和标题。
- 移动端优先保证阅读体验。

视觉关键词：

- 清晰
- 可信
- 游戏感适度
- 资料密度高
- 入口明确

### Phase 3：内容资产建设

目标：形成可长期排名的核心内容。

状态：进行中。

任务：

- 写/改 5-10 篇长期攻略。
- 建立卡组推荐标准格式。
- 建立版本更新标准格式。
- 为重点文章增加 FAQ。
- 新文章统一兼顾公众号阅读体验和 AdSense/搜索质量。
- 视频引用文章不能只嵌播放器，需要补充原创摘要、分析、FAQ 和资料来源。

整改阶段策略：

- 可以正常发布高质量新文章。
- 优先合并同一版本的预告、泄露、预案和正式更新，减少重复搜索意图。

### 11.6 接口与数据库方向
