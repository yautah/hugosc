# 超级细胞营地 2.0 改版 Spec

状态：草案 v0.2  
日期：2026-07-13  
目标：把超级细胞营地从文章流型站点升级为超级细胞游戏中文资料与攻略中心，提升用户价值、搜索质量信号和 AdSense 审核通过率。

## 0. 已确认决策

- 首页定位：使用“Supercell 游戏中文攻略与资料站”，但第一阶段内容权重明显偏向皇室战争。
- URL 结构：新增 `/clashroyale/`、`/brawlstars/`、`/clashofclans/`、`/moco/`、`/tools/` 顶级入口；旧文章 URL 暂不迁移。
- 构建产物：`public/`、`resources/_gen/` 当前存在历史遗留，暂时不作为第一优先级处理。
- Phase 1 优先级：AdSense 相关信任页、薄页面处理、首页/栏目页质量信号优先。
- 视觉方向：资料站/攻略站优先，减少营销感，提升信息密度和导航价值。
- About 页面：已有 `content/about.md`，但内容较薄，需要补强为正式关于页。

## 1. 背景

当前站点主要以 Hugo 文章发布为核心，内容覆盖皇室战争、荒野乱斗、部落冲突、Mo.Co 等 Supercell 游戏。现有结构偏博客流，首页和栏目页对内容价值的组织较弱，Google 可能更容易把它识别为低原创度、薄内容、重复归档页面较多的站点。

超级细胞营地 2.0 的目标不是单纯换 UI，而是重构内容表达方式：

- 从“发布文章”转向“沉淀可长期引用的攻略与资料”。
- 从“最新内容列表”转向“游戏入口 + 专题中心 + 工具/资料入口”。
- 从“搜索引擎抓到大量薄页面”转向“抓到少量但结构清晰的高价值页面”。

## 2. 产品定位

超级细胞营地是面向中文玩家的超级细胞游戏攻略、资料与工具站。

第一阶段以皇室战争为主站点核心，其他游戏保留入口但降低薄页面暴露：

- 皇室战争：主力内容，优先建设栏目、专题、工具和长期攻略。
- 荒野乱斗：保留新闻和攻略入口，后续补齐资料页。
- 部落冲突：保留栏目入口，聚合更新与活动。
- Mo.Co：保留观察和资讯入口。

## 3. 成功指标

短期目标：

- 站点首页能在 10 秒内让新用户理解“这里能帮我解决什么问题”。
- 皇室战争栏目页不再只是文章列表，而是可导航的攻略中心。
- 新文章具备统一的信息增量结构，避免纯资讯感。
- 补齐 AdSense 常见信任页：关于、联系方式、隐私政策、版权声明、免责声明。

中期目标：

- Google Search Console 中低质量/已抓取未编入页面比例下降。
- 核心页面获得更多自然搜索入口。
- 用户从首页进入文章或专题的路径更清晰。

## 4. 信息架构

### 4.1 顶部导航

建议导航：

- 首页
- 皇室战争
- 荒野乱斗
- 部落冲突
- Mo.Co
- 工具
- 关于

说明：

- 不建议把标签、归档、全部分类直接暴露为主要入口。
- “工具”可以先是占位栏目，后续接卡组推荐、活动日历、版本追踪等。

### 4.2 首页

首页从文章流升级为“内容入口页”。

推荐结构：

1. 顶部 Hero
   - 标题：SCBase
   - 副标题：Supercell 游戏中文攻略与资料站
   - 主入口：皇室战争攻略、最新更新、卡组推荐

2. 游戏入口
   - 皇室战争
   - 荒野乱斗
   - 部落冲突
   - Mo.Co

3. 皇室战争核心入口
   - 卡组推荐
   - 版本更新
   - 活动与挑战
   - 新手/进阶指南
   - 卡牌与机制解析

4. 最新攻略
   - 展示高质量文章，不一定按所有内容倒序。
   - 卡片需要包含标题、摘要、更新时间、所属游戏、内容类型。

5. 热门专题
   - 例如：新赛季卡组、成长系统、超级选卡、版本平衡。

6. 站点说明
   - 简短说明本站内容由玩家整理和创作。
   - 放置关于、联系方式、版权声明链接。

### 4.3 皇室战争栏目页

皇室战争栏目页应成为第一阶段最重要的 SEO 和用户入口。

推荐结构：

1. 栏目头部
   - 标题：皇室战争攻略中心
   - 描述：卡组推荐、版本更新、活动挑战、新卡解析与实用指南。

2. 快捷入口
   - 最新卡组
   - 新手指南
   - 版本更新
   - 活动挑战
   - 卡牌解析

3. 当前赛季重点
   - 最新赛季文章
   - 当前强势卡组
   - 当前版本变化

4. 长期指南
   - 如何组卡
   - 卡组原型介绍
   - 超级选卡攻略
   - 新手升级建议

5. 最新文章列表
   - 放在页面后半部分，作为补充。

### 4.4 文章页

文章页保留现有 Hugo Markdown 工作流，但增加标准化内容模块。

建议每篇文章包含：

- 标题
- 更新时间
- 摘要
- 适合人群
- 正文
- 关键结论
- 相关攻略
- FAQ
- 资料来源或说明

Front matter 可逐步扩展：

```yaml
title:
description:
date:
updated:
image:
game: clashroyale
content_type: deck-guide
difficulty: beginner | intermediate | advanced
evergreen: true
featured: false
faq:
  - question:
    answer:
related:
  - /posts/...
```

第一阶段不要求所有旧文章补齐，但新文章应尽量按此结构写。

## 5. 内容类型

### 5.1 长期攻略

长期攻略是超级细胞营地 2.0 的核心质量资产。

示例：

- 皇室战争如何构建一套合理卡组
- 皇室战争卡组原型介绍
- 超级选卡挑战攻略
- 新手升级资源分配指南

要求：

- 能在 3-12 个月后仍然有价值。
- 标题避免纯新闻口吻。
- 提供清晰步骤、判断标准和常见误区。

### 5.2 卡组推荐

卡组推荐要从“列卡组”升级为“解释为什么可用”。

建议结构：

- 适合玩家
- 卡组代码或卡牌列表
- 核心打法
- 起手建议
- 对局思路
- 可替换卡牌
- 优缺点

### 5.3 版本更新

版本更新不只是翻译公告。

建议结构：

- 更新重点
- 对普通玩家的影响
- 强势/弱势变化
- 建议优先做什么
- 相关攻略入口

### 5.4 资讯短文

资讯可以保留，但需要降低站点主体占比。

处理策略：

- 短资讯可以归入周报或专题汇总。
- 避免大量 300-500 字的独立薄页面。
- 无长期价值的旧资讯可考虑 noindex。

## 6. SEO 与 AdSense 改进

### 6.1 必备信任页

需要新增或完善：

- 关于我们
- 联系方式
- 隐私政策
- 版权声明
- 免责声明

### 6.2 薄页面处理

重点检查：

- 空分类页
- 低内容标签页
- 分页页
- 重复归档页
- 未命名页面

处理方案：

- 对低价值标签页加 `noindex,follow`。
- 只保留有真实专题价值的标签入口。
- 分类页增加栏目介绍和精选内容。
- 分页页减少在首页和导航中的权重。

### 6.3 结构化数据

优先添加：

- WebSite
- Organization
- Article
- BreadcrumbList
- FAQPage

### 6.4 内部链接

每篇新文章至少包含：

- 1-3 个相关站内文章链接。
- 1 个栏目页或专题页链接。
- 必要时链接到工具页。

## 7. 视觉与交互方向

整体方向：

- 少营销感，多资料站/攻略站感。
- 首页和栏目页要更像“可探索的目录”，不是单纯博客。
- 卡片信息密度适中，避免只有大图和标题。
- 移动端优先保证阅读体验。

视觉关键词：

- 清晰
- 可信
- 游戏感适度
- 资料密度高
- 入口明确

## 8. Hugo 实现建议

### 8.1 新增模板

建议新增或改造：

- `layouts/index.html` 或主题首页覆盖模板
- `layouts/section/posts.html`
- `layouts/section/clashroyale.html`
- `layouts/partials/content/game-card.html`
- `layouts/partials/content/guide-card.html`
- `layouts/partials/seo/schema.html`

### 8.2 内容数据

可新增：

- `data/games.yaml`
- `data/home.yaml`
- `data/featured.yaml`

示例：

```yaml
clashroyale:
  name: 皇室战争
  description: 卡组推荐、版本更新、挑战攻略与机制解析
  url: /posts/clashroyale/

```

### 8.3 Front matter 兼容

现有字段继续保留：

- `title`
- `image`
- `description`
- `date`
- `slug`
- `categories`
- `tags`
- `keywords`

新增字段应可选，避免一次性迁移所有旧文章。

## 9. 分阶段计划

### Phase 1：基础质量修复

目标：降低 AdSense 低质量风险。

任务：

- 补齐关于、联系方式、隐私政策、版权声明、免责声明。
- 清理或 noindex 低价值标签/分页/归档页。
- 修复首页封面图、卡片信息、栏目入口。
- 为新文章建立写作模板。

### Phase 2：首页与皇室战争栏目页

目标：让用户和搜索引擎看懂站点核心价值。

任务：

- 重做首页信息架构。
- 重做皇室战争栏目页。
- 增加精选攻略和长期指南入口。
- 加强站内链接。

### Phase 3：内容资产建设

目标：形成可长期排名的核心内容。

任务：

- 写/改 5-10 篇长期攻略。
- 建立卡组推荐标准格式。
- 建立版本更新标准格式。
- 为重点文章增加 FAQ。

### Phase 4：工具与数据化页面

目标：形成区别于普通博客的独特价值。

候选工具：

- 皇室战争卡组推荐页
- 当前赛季卡组榜
- 活动/挑战日历
- 卡牌资料页
- 版本更新追踪页

## 10. 第一批建议落地任务

优先级从高到低：

1. 新增基础信任页面。
2. 重做首页为游戏攻略入口页。
3. 重做皇室战争栏目页。
4. 为低价值标签页和分页页加 noindex 策略。
5. 新增文章 archetype，规范新文章结构。
6. 把 3-5 篇已有皇室战争高价值文章改成长期攻略格式。
7. 加 Article、Breadcrumb、FAQ 结构化数据。

## 11. 下一阶段规划

当前基础结构、站点命名和移动端主要卡片布局已初步跑通。下一阶段先做视觉和体验细化，再考虑搜索与数据能力。

### 11.1 Icon 与视觉资产

目标：减少临时感，让站点更像稳定运营的玩家营地。

任务：

- 统一首页、栏目页、工具页里的 icon 风格，避免黑白线稿、游戏素材和站点头像混用时显得割裂。
- 为皇室战争、荒野乱斗、部落冲突、Mo.Co、工具等入口设计稳定的图标规则。
- 检查移动端 icon 尺寸、对齐和留白，确保左 icon 右文案的卡片足够紧凑。
- 评估是否需要一组站内自有图标，减少对游戏官方素材的依赖。

### 11.2 布局与信息密度

目标：让页面更容易扫读，尤其是移动端。

任务：

- 继续压缩首页和栏目页移动端首屏高度，减少重复下滑。
- 细化 Hero、入口卡片、精选攻略、最新文章、footer 的间距节奏。
- 优化首页、皇室战争页、通用游戏频道页之间的组件一致性。
- 检查 footer、工具页、列表页、文章页在窄屏下的换行和视觉重心。
- 保持页面信息密度，不走营销落地页风格。

### 11.3 UI 配色与整体风格

目标：形成“玩家营地”气质，而不是默认模板或纯资料表格感。

方向：

- 主色避免单一蓝色统治全站，可尝试用黑白基础、少量高饱和游戏色作为点缀。
- 皇室战争、荒野乱斗、部落冲突、Mo.Co 可以有轻量区分色，但全站仍保持统一。
- 按钮、标签、卡片边框、hover 状态需要统一一套规则。
- 谨慎使用渐变和大面积装饰，优先保证可读性、速度和内容可信度。

### 11.4 搜索能力

目标：当文章和工具数量增加后，让用户能直接找到卡组、更新、工具或关键词文章。

候选方案：

- 第一阶段：静态站内搜索，基于 Hugo 生成 JSON 索引，前端本地搜索。
- 第二阶段：接入 Pagefind、FlexSearch 或 Fuse.js，支持标题、摘要、标签、游戏分类搜索。
- 第三阶段：如果内容数据化，再考虑服务端搜索或数据库查询。

注意事项：

- 搜索入口不应过早喧宾夺主，先放在 header 或工具页中轻量试用。
- 搜索结果需要展示所属游戏、内容类型、更新时间和摘要。
- 避免把低质量标签页重新暴露成搜索噪音。

### 11.5 接口与数据库方向

目标：为后续工具和资料型页面预留技术路线。

可能场景：

- 皇室战争卡组、卡牌、活动、版本变更等结构化资料。
- 工具页读取接口数据，例如玩家查询、卡组推荐、活动日历。
- 内容和工具之间共享数据，而不是散落在 Markdown 正文里。

阶段建议：

- 先保持 Hugo 静态站为主，数据以 JSON/YAML/TOML 文件进入仓库，方便版本管理和部署。
- 当需要频繁更新、用户查询或第三方 API 聚合时，再引入轻量接口。
- 如果后续接数据库，优先考虑只让工具页面读接口，文章页仍保持静态生成，降低部署复杂度。

开放问题：

- 搜索是否只覆盖文章，还是也覆盖工具、卡组、资料页？
- 数据源优先放在仓库文件、外部 API，还是独立数据库？
- 是否需要为工具页单独拆出前端应用，还是继续嵌在 Hugo 静态页面中？
