# WeChat Template Reference: Redream Obsidian Blue

Source inspected: https://md.redream.cn/

This note captures the WeChat article style currently used manually in md.redream.cn with:

- Font: 无衬线
- Paragraph size: 16px
- Accent color: 经典蓝
- Code theme: obsidian

The site is a Vue single-page app. Its WeChat output is generated by a custom Markdown renderer that applies inline styles to each Markdown element. This reference is an implementation spec for a future `wechat.template` option, not a verbatim copy of the site's bundled code.

## Base Settings

```yaml
wechat:
  template: redream-obsidian-blue
```

Recommended base values:

```text
font-family: -apple-system-font, BlinkMacSystemFont, Helvetica Neue, PingFang SC, Hiragino Sans GB, Microsoft YaHei UI, Microsoft YaHei, Arial, sans-serif
font-size: 16px
line-height: 1.75
text-align: left
text-color: #3f3f3f
accent-color: rgba(15, 76, 129, 1)
link-color: #576b95
```

## Block Elements

`h1`

- Centered, bold, display-table title.
- Margin roughly `2em auto 1em`.
- Horizontal accent underline in classic blue.
- Font size should be about `1.14 * base_size`, so `18.24px` for 16px base.

`h2`

- Centered, bold, display-table section label.
- White text on classic blue background.
- Margin roughly `4em auto 2em`.
- Font size should be about `1.1 * base_size`, so `17.6px` for 16px base.

`h3`

- Bold, left-aligned subsection heading.
- Classic blue left border.
- Tight line height.
- Font size equals base size.

`h4`

- Bold minor heading.
- Classic blue text.
- Font size equals base size.

`p`

- Margin around `1.5em 8px`.
- Letter spacing around `0.1em`.
- Uses base font family, base size, line height, and `#3f3f3f`.

`blockquote`

- No left border.
- Light gray rounded background.
- Padding around `1em`.
- Margin around `2em 8px`.
- Inner paragraph text uses slightly muted text color.

`figure`

- Margin around `1.5em 8px`.

`img`

- Block image.
- Full width.
- Small rounded corners.
- Centered with compact vertical margin.

`figcaption`

- Centered.
- Muted gray.
- Smaller text, about `0.8em`.

`hr`

- Thin light divider.
- Uses scale transform to make the line visually lighter.

`ul` / `ol`

- Left padding around `1em`.
- Compact list item spacing.
- Unordered list marker can render as `•`.
- Ordered list marker can render as `1.` / `2.` inline.

`pre` / code block

- Obsidian highlight.js theme.
- Rounded dark code block.
- Padding around `1em`.
- Line height around `1.5`.
- Horizontal overflow allowed.
- Monospace font stack: Menlo, Operator Mono, Consolas, Monaco, monospace.

## Inline Elements

`strong`

- Bold.
- Classic blue text.

`code`

- Small inline code style.
- Red-ish code color.
- Light gray background.
- Small horizontal padding.
- Rounded corners.

`a`

- For ordinary external links, md.redream.cn renders text as styled spans unless citation mode is enabled.
- WeChat article links keep `#576b95` and no decoration.

`em`

- Italic inline span.

## Renderer Behaviors To Match Later

- Markdown is rendered into inline-style HTML, not class-based CSS.
- Non-WeChat links can optionally become footnote citations.
- Images render as `<figure><img ...><figcaption>...</figcaption></figure>` when alt/caption exists.
- Empty paragraphs are skipped.
- Code block language falls back to plaintext.
- Code block whitespace is preserved with non-breaking spaces.

## Implementation Notes

For content-publisher, implement this as a template layer above Markdown rendering:

```text
Markdown AST / parsed blocks
-> WeChat renderer
-> redream-obsidian-blue style map
-> inline HTML
```

Avoid relying on Hugo theme CSS for this template. The WeChat editor is more predictable with inline styles.
